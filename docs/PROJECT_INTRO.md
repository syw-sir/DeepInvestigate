# DeepInvestigate v4.1 — 项目介绍话术

> 面试开场必问题："介绍一下你觉得最有挑战的项目"
> 目标：2-3 分钟讲清楚，留钩子让面试官追问你准备好的方向

---

## 版本一：标准版（2 分钟，通用）

> "我介绍一个个人项目叫 DeepInvestigate，是一个基于 LangGraph 的多智能体自主安全调查与处置系统。
>
> **它做什么**：用户用自然语言描述一个安全调查任务——比如'帮我检查当前主机有没有安全隐患'——系统会自动规划调查步骤、调用 14 个主机取证工具自主采集进程列表、网络连接、启动项、注册表等，分析后生成结构化报告，甚至可以自动处置威胁。
>
> **架构上**，我用了 LangGraph 搭建了四个 Agent 协作：Planner 负责拆解任务、Investigator 负责调用工具执行调查、Reporter 负责写报告、还有一个 Critic 负责审查报告质量——从 7 个维度打分，不合格就自动打回去补充调查。另外做了一个闲聊快捷路由，简单问候绕过调查流水线直接应答。
>
> **技术上比较有意思的几个点**：
> 一个是 v4.0 的自主取证能力——系统不再等用户喂日志，8 个取证工具直接调用 PowerShell 采集主机信息，包括进程扫描、网络连接、启动项、注册表、Defender 日志等，智能体自己找证据。
> v4.1 加了处置闭环——6 个处置工具（终止进程、隔离文件、封禁 IP、清除启动项、修复注册表、回滚操作），每个操作前自动备份，支持一键回滚。
> 另一个是 HITL 人工审批——处置工具（风险等级 3）调用前自动暂停，前端弹审批卡片，上面有工具名、风险等级、批准/拒绝按钮。这个审批 UI 是我自己加到前端 Next.js 里的。调查工具（风险 0-2）直接放行，确保调查流程不被频繁打断。
> 还踩过一个坑：取证工具在非管理员权限下会失败，比如 Get-NetTCPConnection 需要管理员但我不是。我做了非管理员回退——网络检查改用 netstat -ano，进程扫描去掉 -IncludeUserName，保证在普通用户下也能跑通大部分调查。
> 另外做了 Critic 自审查闭环、多模型路由、RAG 知识库、四层长期记忆、安全护栏这些。
>
> **效果上**，整个系统从'用户说一句话'到'自动出安全报告'全链路跑通，8 个取证工具全部可用，6 个处置工具带备份回滚。项目是全栈的，后端 FastAPI + LangGraph，前端 Next.js 14 + TypeScript。
>
> 如果对哪个部分感兴趣我可以展开讲。"

## 版本二：精简版（1 分钟，快速过）

> "我做了一个 LangGraph 多智能体项目，四个 Agent 协作做自主主机安全调查和威胁处置——Planner 规划、Investigator 调用 23 个工具执行、Reporter 写报告、Critic 审查。
>
> v4.0 实现了 8 个自动化主机取证工具，Agent 自主采集进程/网络/注册表等信息；v4.1 加了 6 个处置工具，带自动备份和一键回滚。
>
> 亮点是做了 HITL 人工审批——处置操作自动拦截，前端弹确认卡片带批准/拒绝按钮；非管理员环境下用 netstat 等回退方案保证基本调查可用。
>
> 还集成了 RAG、长期记忆、多模型路由、安全护栏。全栈 FastAPI + Next.js。"

## 版本三：技术深度版（3 分钟，适合面试官明显是技术专家）

> "我介绍一个 LangGraph 的多智能体项目 DeepInvestigate，版本已经迭代到 v4.1。
>
> **背景**：我做这个项目的初衷是想做一个真正'自主'的智能体——不是等用户喂数据，而是自己去找证据。安全调查场景天然适合验证这个能力：需要多步骤推理、需要实际执行系统命令、需要根据结果动态调整调查方向。
>
> **架构**：四个 Agent 共享 messages 历史但通过 system prompt 隔离职责。Planner 温度 0.3 保证规划稳定，Investigator 温度 0.4 平衡探索，Reporter 温度 0.5 让报告自然，Critic 温度 0.2 保证审查一致。另外做了闲聊检测——'你好'这类简单问候直接走快捷路由，不经过完整调查流水线。
>
> **v4.0 自主取证**：8 个取证工具全部用 Python 封装 PowerShell 命令。进程扫描用 Get-Process 不用 CIM/WMI（那台机器 WMI 会卡死），网络检查用 netstat -ano 做非管理员回退，启动项检查遍历注册表 Run 键 + 计划任务 + 自启服务。每个工具独立模块，通过 registry.py 统一注册。
>
> **v4.1 处置闭环**：6 个处置工具——kill_process / quarantine_file / block_ip / remove_startup / repair_registry / rollback_action。每个操作执行前自动备份（进程信息存 JSON、注册表导出 .reg、计划任务导出 XML），rollback_action 支持任意操作回滚。处置工具风险等级 3，必须 HITL 审批。
>
> **HITL 实现**：工具分三级风险——只读查询 Level 0 直接放行，取证工具 Level 1-2 自动放行，处置工具 Level 3 触发 interrupt()。前端 SSE 流中检测到 '需要人工审批' 标记后停止当前流、展示审批卡片（工具名 + 风险等级 + 批准/拒绝按钮）。点批准后调 /chat/resume 继续流式输出。这里踩过一个坑——一开始用 interrupt_before 全局拦截，发现安全工具也被卡了，后来改成在节点内部用 interrupt() 按风险等级条件触发。
>
> **Critic 闭环**：审查 7 个维度——事实准确性、逻辑完整性、证据充分性、关键信息缺失、建议可行性、处置安全性、处置完整性。评分低于 4 分自动回退，最多 2 次防止无限循环。回退时把审查建议注入上下文，Agent 增量补充而非重做。
>
> **其他**：SQLite Checkpointer 做会话持久化和时间旅行调试；SSE 适配器将 LangGraph 事件流映射为 OpenAI 兼容 chunk，附带 final_answer 补偿机制；多模型路由按角色分配加熔断降级；工具封装支持 MCP 协议。
>
> **评估**：自研评估体系，10 个指标，按难度和类别分组，含对抗样本。每次改动跑回归出对比报告。
>
> 整个项目让我理解了 Agent 系统从 Demo 到生产之间的差距——很多问题不是'能不能做'而是'挂了怎么办'。比如取证工具在非管理员机器上怎么优雅降级、HITL 审批流在前端怎么交互、递归上限怎么设置才不炸。"

## 关键钩子设计

介绍完项目后，面试官通常会追问。你要在介绍中有意留钩子，引导面试官问你准备好的方向：

| 你说的话 | 面试官大概率会追问 |
|---|---|
| "踩过一个坑，从 `interrupt_before` 改到 `interrupt()`" | "这两个有什么区别？" → 接 Q4 |
| "Critic 评分不合格就自动打回去" | "会不会无限循环？" → 接 Q4 |
| "非管理员下取证工具会失败，我做了回退" | "怎么检测权限的？回退策略是什么？" → 接 Q8 |
| "前端 SSE 流中检测 HITL 标记，弹审批卡片" | "前后端怎么联动的？" → 接 Q5 |
| "工具分三级风险，Level 3 才触发审批" | "怎么分级的？拒绝后 Agent 怎么办？" → 接 Q3/Q10 |
| "v4.1 处置操作自动备份，支持回滚" | "回滚粒度是什么？备份存在哪？" → 接 Q19 |
| "自研了评估体系" | "怎么评估的？幻觉怎么检测？" → 接 Q16/Q17 |

**策略**：不要一次性把所有细节倒出来。介绍时点到为止，等面试官追问再展开。这样对话是互动的，不是背稿。

## 常见追问及快速应答

### "你为什么选安全调查这个场景？"

> "两个原因：一是安全调查天然需要多步骤推理和真实工具调用，适合验证 Agent 架构的自主性——不是调个天气 API 写首诗，而是实际执行系统命令、分析结果、动态调整策略；二是主机取证用的 PowerShell 命令本身就是结构化的，方便封装成工具。"

### "这个项目有人用吗？"

> "目前是个人项目，主要用来验证 Agent 架构设计和准备面试。但架构上考虑了生产化——有 Checkpointer 持久化、HITL 审批、多模型 fallback、审计日志、处置回滚，不是 Demo 级别的。"

### "你一个人做的？做了多久？"

> "对，个人项目。v1 是单 Agent，v2 上了 LangGraph 多 Agent，v3 加了 HITL、Critic、MCP，v4 做了 14 个主机取证/处置工具和前端审批 UI。迭代了大概两个多月，主要时间花在踩坑和调架构上。"

### "遇到的最大挑战是什么？"

> "最大的挑战不是功能实现，而是**让系统在异常情况下不崩**。比如取证工具在非管理员机器上 WMI 卡死怎么回退、PowerShell 命令里的大括号被 Python .format() 吃掉怎么修、递归上限不够导致调查中断怎么调整。这些边界情况的处理比正常流程复杂得多。"

## 面试中的肢体语言提示

- 介绍架构时，可以**用手比划流程**：从左到右 Planner → Investigator → Reporter → Critic
- 说到"踩坑"时**笑一下**，显得真实而不是背稿
- 说到"如果感兴趣我可以展开"时**停顿一下**，给面试官插话的机会
- 不要盯着简历念，**看着面试官说**

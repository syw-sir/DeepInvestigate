# DeepInvestigate v3.0 — 项目介绍话术

> 面试开场必问题："介绍一下你觉得最有挑战的项目"
> 目标：2-3 分钟讲清楚，留钩子让面试官追问你准备好的方向
---

## 版本一：标准版（2 分钟，通用）

> "我介绍一个个人项目叫 DeepInvestigate，是一个基于 LangGraph 的多智能体安全调查系统。
>
> **它做什么**：用户用自然语言描述一个安全调查任务——比如'分析这段 SSH 日志有没有暴力破解'——系统会自动规划调查步骤、调用工具去执行、最后生成结构化报告。
>
> **架构上**，我用了 LangGraph 搭建了四个 Agent 协作：Planner 负责拆解任务、Investigator 负责调用工具执行调查、Reporter 负责写报告、还有一个 Critic 负责审查报告质量，不合格就自动打回去补充调查。
>
> **技术上比较有意思的几个点**：
> 一个是做了 HITL 人工审批——危险工具比如代码执行、文件写入，调用前会自动暂停弹审批卡片，安全工具直接放行。这里踩过一个坑，一开始用 LangGraph 的 `interrupt_before` 全局拦截，发现安全工具也被卡了，后来改成在节点内部用 `interrupt()` 按风险等级条件触发。
> 另一个是做了多模型路由——不同 Agent 角色分配不同模型，主模型挂了自动 fallback，还带了 Token 成本追踪。
> 还集成了 RAG 知识库、四层长期记忆、安全护栏这些。
>
> **效果上**，我搭了一套评估体系，10 个指标，每次改代码跑回归看有没有退化。整个项目是全栈的，后端 FastAPI，前端 Next.js。
>
> 如果对哪个部分感兴趣我可以展开讲。"

## 版本二：精简版（1 分钟，快速过）

> "我做了一个 LangGraph 多智能体项目，四个 Agent 协作做安全日志调查——Planner 规划、Investigator 执行、Reporter 写报告、Critic 审查。
>
> 亮点是做了 HITL 人工审批，危险工具自动拦截等用户确认；还有多模型路由加 fallback、SQLite 持久化支持断点续跑。
>
> 集成了 RAG、长期记忆、安全护栏，自研了评估体系做回归测试。全栈 FastAPI + Next.js。"

## 版本三：技术深度版（3 分钟，适合面试官明显是技术专家）

> "我介绍一个 LangGraph 的多智能体项目 DeepInvestigate。
>
> **背景**：安全调查场景天然适合多 Agent——任务需要规划、执行、报告三个不同能力，单 Agent 上下文容易污染，不同阶段需要不同推理温度。
>
> **架构**：四个 Agent 共享 messages 历史但通过 system prompt 隔离职责。Planner 温度 0.3 保证规划稳定，Investigator 温度 0.4 平衡探索，Reporter 温度 0.5 让报告自然，Critic 温度 0.2 保证审查一致。
>
> **HITL 实现**：工具分三级风险——只读查询 Level 0 直接放行，代码执行 Level 1、日志查询 Level 2 触发 `interrupt()`。这里从 `interrupt_before` 改到 `interrupt()` 的过程让我深入理解了 LangGraph 的中断机制——前者是编译时静态的，后者是运行时动态的，可以做条件判断。
>
> **Critic 闭环**：审查四个维度——事实准确性、逻辑完整性、信息完整度、建议可行性。评分低于 4 分自动回退，最多 2 次防止无限循环。回退时把审查建议注入上下文，Agent 增量补充而非重做。
>
> **其他**：SQLite Checkpointer 做会话持久化和时间旅行调试；多模型路由按角色分配加熔断降级；工具封装成 MCP Server 支持跨框架复用；安全护栏做输入拦截加 PII 脱敏。
>
> **评估**：10 个指标，按难度和类别分组，含对抗样本。每次改动跑回归出对比报告。
>
> 整个项目让我理解了 Agent 系统从 Demo 到生产之间的差距——很多问题不是'能不能做'而是'挂了怎么办'。"

## 关键钩子设计

介绍完项目后，面试官通常会追问。你要在介绍中有意留钩子，引导面试官问你准备好的方向：

| 你说的话 | 面试官大概率会追问 |
|---|---|
| "踩过一个坑，从 `interrupt_before` 改到 `interrupt()`" | "这两个有什么区别？" → 接 Q4 |
| "Critic 评分不合格就自动打回去" | "会不会无限循环？" → 接 Q4 |
| "不同 Agent 用不同温度" | "为什么这么分？" → 接 Q2 |
| "工具分三级风险" | "怎么分级的？拒绝后 Agent 怎么办？" → 接 Q3/Q10 |
| "自研了评估体系" | "怎么评估的？幻觉怎么检测？" → 接 Q16/Q17 |

**策略**：不要一次性把所有细节倒出来。介绍时点到为止，等面试官追问再展开。这样对话是互动的，不是背稿。

## 常见追问及快速应答

### "你为什么选安全调查这个场景？"

> "两个原因：一是安全调查天然需要多步骤推理和工具调用，适合验证 Agent 架构；二是我对安全领域有兴趣，MITRE ATT&CK 和 CVE 是公开的结构化知识，方便做 RAG。"

### "这个项目有人用吗？"

> "目前是个人项目，主要用来验证 Agent 架构设计和准备面试。但架构上考虑了生产化——有 Checkpointer 持久化、HITL 审批、多模型 fallback、审计日志，不是 Demo 级别的。"

### "你一个人做的？做了多久？"

> "对，个人项目。v1 是单 Agent，v2 上了 LangGraph 多 Agent，v3 加了 HITL、Critic、MCP 这些生产级特性。迭代了大概两个月，主要时间花在踩坑和调架构上。"

### "遇到的最大挑战是什么？"

> "最大的挑战不是功能实现，而是**让系统在异常情况下不崩**。比如工具调用失败了 Agent 怎么恢复、Critic 误判了怎么防止无限回退、API 挂了怎么降级。这些边界情况的处理比正常流程复杂得多。"

---

## 面试中的肢体语言提示

- 介绍架构时，可以**用手比划流程**：从左到右 Planner → Investigator → Reporter → Critic
- 说到"踩坑"时**笑一下**，显得真实而不是背稿
- 说到"如果感兴趣我可以展开"时**停顿一下**，给面试官插话的机会
- 不要盯着简历念，**看着面试官说**
